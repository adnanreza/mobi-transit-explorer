# Feature 012 - Accessibility and Responsive Polish

Branch: `feature/accessibility-responsive-polish`

## Goal

Improve accessibility and responsive quality before public deployment.

## Scope

Review and polish the full MVP.

## Requirements

Check semantic headings, button labels, focus states, keyboard accessibility, color contrast, mobile layout, tablet layout, reduced clutter on smaller screens, readable tables, and useful empty states.

## Tests

- Key headings.
- Labelled controls.
- Important buttons.
- Accessible names where practical.

## Acceptance Criteria

- App is usable on desktop.
- App is acceptable on mobile.
- Interactive controls are understandable.
- No obvious accessibility issues remain.
