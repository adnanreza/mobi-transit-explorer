# Feature 013 - Portfolio README and Case Study Polish

Branch: `feature/readme-case-study-polish`

## Goal

Make the repository read like a strong portfolio piece.

## Scope

Polish:

- `README.md`
- `docs/product-spec.md`
- `docs/data-methodology.md`
- `docs/review-checklist.md`

## README Must Include

- Project title.
- Short description.
- Future live URL.
- Stack.
- Feature list.
- Screens/sections.
- Data methodology.
- Feature lifecycle.
- Setup instructions.
- Test instructions.
- Future improvements.
- Note that MVP uses mock data.

## Portfolio Framing

Emphasize front-end product thinking, data-product UI, urban mobility interest, Mobi/transit connection, and future public data integration.

## Tests

Run the full test suite.

## Acceptance Criteria

- README is clear to a recruiter or hiring manager.
- Docs explain why the project exists.
- Project looks intentional, not randomly generated.
